Foundation Runtime is a set of modules that we designed at Cisco for server applications. We thought they could be helpful to the broader community so we open sourced them. The modules stand by themselves, but are most helpful when used together to provide a holistic solution to standard needs of enterprise server systems. The main modules are:

  • Common Configuration
  • Communication API - this includes HTTP client and Server, Async Messaging
  • Logging (including structured logging)
  • Service Discovery infrastructure
  • Monitoring
  • Other goodies

There’s nothing new in an HTTP client module or a logging library; the point is that these disparate areas of plumbing are integrated together using best of breed 3rd party open source software (Apache HttpClient, SLF4J, Jetty Server - to name a few) with the following powerful advantages:

  • Smart defaults
  • Client side load balancing
  • Structured logging
  • Automatic correlation ids in log files to assist tracking a single transaction from different processes' logs
  • Asynchronous Messaging
  • Integrated Service Directory
  • Central Configuration

More info in the detailed wikis. And we’ll answer questions too.
